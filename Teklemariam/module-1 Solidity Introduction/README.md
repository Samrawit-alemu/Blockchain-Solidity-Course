# Module 1 - Solidity Introduction

## Overview
This module introduces the fundamentals of Solidity and smart contract development. You will learn Solidity data types, function declarations, visibility keywords, and the differences between `view` and `pure` functions.

## What i learn
- Basic Solidity syntax and contract structure
- Boolean values and logical expressions
- Unsigned and signed integer types
- String literals and enum types
- Writing functions with arguments and return values
- Using `view` and `pure` functions
- Function overloading and console logging for debugging

## How to use
1. Open the contract file in each lesson.
2. Read the instructions inside the contract and complete the missing logic.
3. Run tests using Forge or your preferred Solidity toolchain.

## Notes
- `view` functions read state without modifying it.
- `pure` functions do not read or modify state.
- `console.log` is useful during local testing, but not for production deployment.
