# Naol - Module 3: Reference Types and Storage

## What this module teaches
This module covers Solidity reference types and how data is stored and accessed in smart contracts.

## Highlights
- Working with fixed and dynamic arrays
- Differences between `storage` and `memory`
- Filtering values and returning results from arrays
- Using mappings for fast lookup
- Structuring related data with structs
- Nested mappings and advanced data models

## Structure
- `Arrays/`
- `Mappings/`
- `Structs/`

## How to use it
1. Open each exercise and identify the intended data structure.
2. Complete the contract logic to store or retrieve values.
3. Compare how each type behaves in tests.

## Note
The focus here is on choosing the right reference type for each problem.
