# Naol - Module 4: Applied Solidity Patterns

## What this module teaches
This module applies Solidity concepts to build reusable contracts through inheritance and a voting system.

## Highlights
- Inheriting behavior from parent contracts
- Passing constructor parameters in derived contracts
- Overriding functions using `virtual` and `override`
- Coordinating parent and child logic with `super`
- Adding owner-based permission checks
- Creating and executing proposal-based voting
- Logging contract activity with events

## Structure
- `Inheritance/`
- `Voting/`

## How to use it
1. Review each contract example to understand the design pattern.
2. Implement missing inheritance or voting logic.
3. Validate behavior with tests and event checks.

## Note
This module is designed to bridge basic Solidity knowledge and real contract architecture.
